Project Overview

Clyo is an innovative social communication platform aiming to combine the best elements of apps like WhatsApp, Snapchat, Instagram, and Facebook. Our main goal is to build a secure and privacy-focused messenger with end-to-end encryption (E2EE), offering rich communication options within group chats and communities (Spaces).

Frontend (Flutter App - `app/`)

  • ‚úì Status: The application is developed in Flutter, ensuring multi-platform availability (Android, iOS, Web, Desktop).
  • ‚úì Key Areas: Strong emphasis is placed on end-to-end encryption (E2EE) using libraries like `pointycastle`, ensuring maximum user privacy.
  • ‚úì Communication Model: A dual chat model is implemented ‚Äì Groups for small, private conversations and Spaces for larger communities with multiple rooms (`text`, `voice`, `post`).
  • ‚úì Ongoing Work: Actively working on refining the user interface and stability for existing features like text and voice messages. We had pending UI for attachments (photos, files), which is now fully implemented.
  • ‚úì Future Expansions: The code already contains hints of future features such as a social "feed" (`feed_screen.dart`) and location sharing (`map_screen.dart`), signaling our vision for a broader social network.

Backend (Deno, TypeScript, Hono - `server/`)

  • ‚úì Status: The backend is built on modern Deno, TypeScript, and Hono technologies, guaranteeing high performance and reliability.
  • ‚úì Database: We use PostgreSQL with Prisma ORM for robust data management.
  • ‚úì Authentication: In addition to standard JWT authentication, modern passkey authentication (`@simplewebauthn/server`) is implemented, enhancing security and user convenience.
  • ‚úì Services: The backend provides APIs for user, message, group, space, and media file management. Key components also include services for push notifications (FCM) and WebSocket communication for real-time messages.
  • ‚úì Maintenance: Regular background tasks are performed, such as cleaning up old messages (`cron/cleanup-messages.ts`), to maintain efficiency and database size.
